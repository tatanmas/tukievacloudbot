# 2026-03-17

## Incidente: respuesta cortada tras exec en chat gateway

- **Qué pasó:** En el chat del dashboard (127.0.0.1:18789/chat, sesión agent:main), el agente ejecutó "Ver mis tareas activas en contexto Eva", mostró "Completed" y **no** envió mensaje en lenguaje natural después. La conversación quedó cortada.
- **Revisión de logs (causa a nivel ejecución):**
  - Run **19:43:12** (4:43 PM local): `embedded run start` runId=fd6fc211, messageChannel=webchat. **Exec** a las 19:43:28–19:43:29. **Run completed** a las 19:43:57 (durationMs=44691, **aborted=false**). El gateway terminó el run con normalidad.
  - A las **19:44:21** el **webchat se desconectó** con **code=1001** (Going Away), conn=1f34ee84 — la misma conexión del chat.
  - Conclusión: el servidor **sí** generó y completó el run; no hubo abort ni error del agente. La causa del “corte” a nivel ejecución es **desconexión del cliente (WS 1001)** ~24 s después de que el run terminara: o bien la UI cerró/refrescó la conexión y dejó de recibir el resto del stream, o la Control UI no está mostrando el contenido que llega **después** del bloque exec.
- **Acciones:** Reforzamos en AGENTS.md la regla "Después de cada herramienta: siempre responder". Para evitar cortes: no refrescar/cambiar de pestaña mientras responde el agente; si persiste, revisar en la Control UI si el mensaje se corta al recibir un bloque de tool/exec.

## Incidente: mensaje largo sin respuesta (5:35–5:36 PM)

- **Qué pasó:** El usuario envió un mensaje (posiblemente largo; historial de chat también largo con muchos mensajes de Juan Trujillo). El asistente no mostró respuesta; en la UI solo aparecen timestamp A 5:35 p.m. y U 5:36 p.m. sin contenido.
- **Logs:** Run **20:35:00** (webchat) inició y **completó en 1,3 s** (durationMs=1328, aborted=false). Un run normal tarda decenas de segundos; 1,3 s indica que el modelo casi no procesó (contexto rechazado, mensaje truncado o respuesta vacía). A las 20:35:27 webchat desconectó (1001) y reconectó.
- **Causa probable:** Conversación + mensaje nuevo superaron el contexto útil o el límite aceptado; la API o el gateway devolvieron/truncaron sin error explícito en log.
- **Acciones:** (1) Activado **contextPruning** en `.clawdbot/clawdbot.json` con `mode: "rolling"` para recortar resultados de tools y aliviar contexto. (2) Documentado en TOOLS.md: para mensajes muy largos o hilos largos, usar **Nueva sesión** o resumir el mensaje. (3) Si sigue fallando, probar sesión nueva antes de enviar el mensaje largo.
